# 📋 Project: Ice Breaker RE - 产品需求文档 (PRD)

**文档版本：** v1.0  
**创建日期：** 2025-11-21  
**项目状态：** 原型阶段 → 垂直切片阶段  
**目标平台：** PC (Windows/Mac/Linux) + 移动端 (iOS/Android) + 主机 (Switch/PS5/Xbox)

---

## 一、项目概述

### 1.1 项目定位

**项目名称：** Ice Breaker RE  
**项目类型：** 垂直卷轴平台跳跃 + Roguelite + 多人合作/竞技  
**目标受众：** 
- 核心玩家：喜欢挑战性平台游戏的玩家（18-35岁）
- 休闲玩家：喜欢聚会游戏的玩家（全年龄段）
- 怀旧玩家：FC时代《敲冰块》的粉丝

### 1.2 核心价值主张

1. **保留经典手感：** 冰面滑行、头部撞击破坏的核心玩法
2. **现代化改进：** Coyote Time、角落辅助等优化，减少挫败感
3. **次世代画质：** Ray Tracing、SSS等先进渲染技术
4. **无限重玩：** Roguelite程序化生成，每次都是新体验
5. **社交乐趣：** 双人互坑机制，聚会游戏首选

### 1.3 项目目标

**短期目标（3个月）：**
- 完成Python原型，验证核心玩法
- 实现手感优化（Coyote Time、角落辅助）
- 完成Unity垂直切片，验证视觉目标

**中期目标（6个月）：**
- 完成Unity完整产品开发
- 实现程序化关卡生成
- 完成多人系统

**长期目标（12个月）：**
- 多平台发布（PC、移动端、主机）
- 持续内容更新（新Biome、新道具）
- 社区建设和玩家反馈迭代

## 二、功能需求

### 2.1 核心玩法功能

#### 2.1.1 玩家控制系统

**需求ID：** FR-001  
**优先级：** P0 (必须)

**功能描述：**
- 玩家可以使用方向键/WASD控制角色左右移动
- 玩家可以按空格键/跳跃按钮进行跳跃
- 角色在冰面上具有滑行惯性（低摩擦力）
- 角色跳跃轨迹为抛物线，空中可进行有限调整（20%空中控制）

**验收标准：**
- [ ] 角色移动流畅，无卡顿
- [ ] 冰面滑行感明显（摩擦力系数 < 0.1）
- [ ] 跳跃手感舒适，支持Coyote Time（0.15秒窗口）
- [ ] 空中控制有限，不会破坏挑战性

**技术实现：**
- 使用Unity Character Controller或Rigidbody
- 物理参数：摩擦力 -0.08，重力 0.8，跳跃力度 -16

---

#### 2.1.2 头部撞击破坏系统

**需求ID：** FR-002  
**优先级：** P0 (必须)

**功能描述：**
- 玩家跳跃时头部撞击冰块，冰块会被破坏
- 破坏时产生物理碎片和粒子效果
- 碎片可以掉落并影响下方物体（敌人、玩家）

**验收标准：**
- [ ] 头部撞击判定准确，无误判
- [ ] 冰块破坏效果明显（粒子+碎片）
- [ ] 碎片具有物理碰撞，可以砸到敌人
- [ ] 破坏音效和屏幕震动反馈

**技术实现：**
- 使用Unity Physics进行碰撞检测
- 使用Fracture插件或自定义破碎算法
- 使用VFX Graph创建粒子效果

---

#### 2.1.3 角落辅助系统

**需求ID：** FR-003  
**优先级：** P1 (重要)

**功能描述：**
- 当玩家头部接近冰块边缘时，自动将玩家"挤"到空隙处
- 避免玩家因为微小偏差而无法攀爬

**验收标准：**
- [ ] 边缘检测准确（10像素容差）
- [ ] 自动调整流畅，不突兀
- [ ] 不影响正常破坏机制

**技术实现：**
- 在碰撞检测中添加边缘判断逻辑
- 使用Vector2.Lerp平滑调整位置

---

### 2.2 关卡系统

#### 2.2.1 程序化关卡生成

**需求ID：** FR-010  
**优先级：** P0 (必须)

**功能描述：**
- 无限向上生成的垂直关卡
- 每10层为一个Biome（生态区）
- 每个Biome有独特的视觉风格和难度曲线
- 关卡布局随机但保证可玩性（至少一条可行路径）

**验收标准：**
- [ ] 关卡生成速度快（< 100ms）
- [ ] 生成的关卡100%可通关
- [ ] 不同Biome视觉差异明显
- [ ] 难度曲线合理（逐渐增加）

**技术实现：**
- 使用种子（Seed）控制随机性
- 使用A*算法验证路径可达性
- 使用ScriptableObject存储Biome配置

---

#### 2.2.2 垂直卷轴系统

**需求ID：** FR-011  
**优先级：** P0 (必须)

**功能描述：**
- 相机跟随玩家向上移动
- 屏幕下方的关卡自动销毁（内存管理）
- 屏幕上方预生成关卡（无缝体验）

**验收标准：**
- [ ] 相机跟随流畅，无抖动
- [ ] 内存使用稳定（无内存泄漏）
- [ ] 关卡加载无卡顿

**技术实现：**
- 使用Unity Cinemachine进行相机控制
- 使用对象池管理关卡块
- 使用异步加载避免卡顿

---

### 2.3 敌人系统

#### 2.3.1 基础敌人AI

**需求ID：** FR-020  
**优先级：** P1 (重要)

**功能描述：**
- **雪怪（Topi）：** 基础敌人，左右巡逻，会填补玩家破坏的冰块
- **冰刺陷阱：** 静态陷阱，周期性弹出
- **冰霜翼龙：** 飞行敌人，会俯冲抓取玩家

**验收标准：**
- [ ] 敌人行为符合设计（巡逻、攻击、填补）
- [ ] AI性能良好（不影响帧率）
- [ ] 敌人可以被玩家攻击击败

**技术实现：**
- 使用Unity NavMesh或自定义AI状态机
- 使用ScriptableObject配置敌人属性

---

#### 2.3.2 Boss战系统

**需求ID：** FR-021
**优先级：** P1 (重要)

**功能描述：**
- 每10层出现Boss（冰岩巨人）
- Boss有3个阶段，难度递增
- 玩家需要利用"反弹冰柱"机制击败Boss

**Boss阶段化机制：**
- **阶段1（0-50%血量）：** 基础攻击模式，周期性砸地产生冰刺，移动缓慢
- **阶段2（50-25%血量）：** 增加冰柱投掷攻击，冰刺密度提升，移动速度加快
- **阶段3（25-0%血量）：** 狂暴模式，连续攻击，召唤小冰怪辅助，冰刺覆盖全屏

**验收标准：**
- [ ] Boss战有挑战性但不令人沮丧
- [ ] 阶段转换明显（视觉+音效）
- [ ] 反弹机制流畅

**技术实现：**
- 使用状态机管理Boss阶段
- 使用物理系统实现反弹机制

---

#### 2.3.3 威胁梯度系统

**需求ID：** FR-022
**优先级：** P1 (重要)

**功能描述：**
- 敌人根据距离玩家和环境因素动态调整威胁等级
- 威胁梯度从低到高：巡逻 → 警戒 → 攻击 → 狂暴
- 影响敌人的移动速度、攻击频率和伤害输出

**威胁等级定义：**
- **等级1（巡逻）：** 正常巡逻，玩家距离 > 20米
- **等级2（警戒）：** 减速移动，玩家距离 10-20米
- **等级3（攻击）：** 主动接近，玩家距离 5-10米
- **等级4（狂暴）：** 全力攻击，玩家距离 < 5米

**验收标准：**
- [ ] 威胁等级转换流畅，无突兀感
- [ ] 敌人行为符合当前威胁等级
- [ ] 性能影响最小（< 5% CPU占用）

**技术实现：**
- 使用距离检测和视野锥判断威胁等级
- 状态机平滑过渡，避免瞬间切换
- 使用DOTween实现行为插值

---

### 2.4 Roguelite系统

#### 2.4.1 热量值收集系统

**需求ID：** FR-030  
**优先级：** P1 (重要)

**功能描述：**
- 玩家收集关卡中的"蔬菜/水果"获得热量值
- 热量值可以在每层休息区购买道具
- 热量值在死亡时清零（Roguelite机制）

**验收标准：**
- [ ] 收集物视觉明显，易于识别
- [ ] 收集反馈良好（音效+粒子）
- [ ] 热量值显示清晰

**技术实现：**
- 使用Unity UI系统显示热量值
- 使用ScriptableObject配置收集物属性

---

#### 2.4.2 道具商店系统

**需求ID：** FR-031  
**优先级：** P1 (重要)

**功能描述：**
- 每10层（Biome切换）出现商店
- 玩家可以用热量值购买临时道具
- 道具效果持续到下次商店或死亡

**道具列表：**
- **登山钉鞋：** 减少20%打滑 + 增加10%跳跃高度
- **弹力锤：** 攻击距离增加 + 可反弹一次投射物
- **暖宝宝：** 抵挡一次冻结攻击
- **防滑鞋套：** 增加15%摩擦力（减少滑行）

**验收标准：**
- [ ] 商店界面清晰易用
- [ ] 道具效果明显
- [ ] 道具平衡性良好（不会破坏核心挑战）

**技术实现：**
- 使用Unity UI系统创建商店界面
- 使用ScriptableObject配置道具属性
- 使用组件系统实现道具效果

### 2.5 多人系统

#### 2.5.1 本地多人模式

**需求ID：** FR-040

**验收标准：**
- [ ] 幽灵机制有趣，不会让死亡玩家无聊
- [ ] 平衡性良好（不会过于强大）
- [ ] 视觉反馈清晰（半透明效果）

**技术实现：**
- 使用Shader实现半透明效果
- 使用状态机管理幽灵行为
- 使用Unity UI显示模式选择

## 五、技术架构

### 5.1 技术栈

**引擎：** Unity 2022.3 LTS  
**渲染管线：** HDRP (PC) / URP (移动端)  
**物理引擎：** Unity Physics (Havok)  
**网络：** Unity Netcode / Photon (在线多人)  
**音频：** Unity Audio System  
**UI：** Unity UI (uGUI)

### 5.2 项目结构

```
IceBreakerRE/
├── Assets/
│   ├── Scripts/
│   │   ├── Core/           # 核心逻辑
│   │   ├── Player/         # 玩家系统
│   │   ├── Enemy/          # 敌人系统
│   │   ├── LevelGen/       # 关卡生成
```

### 5.3 技术实现细节

#### 5.3.1 Shader Graph 实现

**冰面材质 Shader：**
- 使用 Subsurface Scattering (SSS) 节点实现冰的次表面散射效果
- 结合 Fresnel 节点创建冰面反射
- 利用 Noise 节点添加冰面纹理细节
- 支持动态光照和阴影

**粒子效果 Shader：**
- 自定义粒子材质，支持透明度和颜色渐变
- 使用 Time 节点实现动画效果
- 结合 UV 动画节点创建流动效果

#### 5.3.2 VFX Graph 实现

**冰块破碎粒子系统：**
- 使用 Spawn 节点控制碎片生成数量和速度
- 利用 Physics 模块实现重力影响
- 结合 Lifetime 节点控制粒子存活时间
- 支持碰撞检测和反弹效果

**天气效果系统：**
- 雪花粒子：使用 Billboard 渲染模式，随机速度和旋转
- 暴风雪：结合风力场和粒子发射器
- 极光：使用渐变色和波形动画

#### 5.3.3 Fracture 插件实现

**冰块破碎系统：**
- 使用 Voronoi Fracture 算法生成自然破碎效果
- 支持多级破碎（大块→小块→碎片）
- 结合物理引擎实现实时破碎
- 优化性能：预计算破碎模式，运行时实例化

**破坏触发机制：**
- 头部碰撞检测使用 Capsule Collider
- 冲击力计算基于玩家速度和质量
- 支持连锁反应：相邻冰块自动破碎
